# ロール別権限マトリクス

## ロール定義

| ロール | 値 | 説明 |
|--------|-----|------|
| 生徒 | `student` | 各校在籍生徒 |
| 図書委員 | `student`（`is_committee: true`） | 生徒の権限＋他生徒への代理操作権限 |
| 司書 | `librarian` | 各校図書館担当者 |

## 補足ルール

- `committee`は`student`ロールかつ`is_committee: true`のユーザーを指す
- `librarian`は**自校の資料・貸出のみ**操作可能（他校の資料への操作不可）
- `librarian`は課題1では自身の貸出・予約は行わない（代理操作専任）
- `admin`は全校のアカウントを管理可能
- `student` / `committee`は**自分または代理対象の生徒の貸出・予約のみ**閲覧・操作可能
- パスワードリセットは未ログイン状態で実行するため全ロール対象（認証不要）
- 図書館間貸出の返却処理は通常の返却（UC-06）と同じフローで行う
- 将来的には`is_committee`フラグを権限テーブル方式に移行することを課題2で検討する
- **管理者アカウントの管理方針（課題1）**：管理者アカウントはseed.sqlで初期投入する。追加が必要な場合はDB直接操作で対応する。管理者アカウントの管理画面は課題2の改善提案として検討する

## 貸出ルール

| ルール | 内容 |
|--------|------|
| 貸出上限 | 1人あたり同時5冊まで（図書館間貸出を含む） |
| 延滞制限 | 延滞中（返却期限超過）の資料がある場合、新規貸出・予約不可 |
| 制限の適用 | 代理操作時も同様に適用。フロント表示制御＋バックエンドAPI検証の両方で担保 |
