# 機能一覧

## 課題1（必須）機能

### 生徒機能（`student`）

| 機能ID | 機能名 | 概要 | 対応UC |
|--------|--------|------|--------|
| F-S01 | ログイン | メールアドレス・パスワードでログインする | UC-01 |
| F-S02 | 資料検索 | キーワード・NDC・ISBN等で全校蔵書を横断検索 | UC-02 |
| F-S03 | 資料予約（自校蔵書） | 利用可能な自校資料に予約を入れる | UC-03 |
| F-S04 | 順番待ち登録 | 貸出中資料にキュー登録する | UC-04 |
| F-S05 | 資料貸出 | SCR-16（貸出処理画面）でバーコードをスキャンし自身の貸出を確定する | UC-05 |
| F-S06 | 資料返却 | SCR-17（返却処理画面）でバーコードをスキャンし自身の返却を確定する | UC-06 |
| F-S07 | パスワード変更 | 自分のパスワードを変更する | UC-07 |
| F-S08 | パスワードリセット申請 | メールアドレスを入力しリセット用URLを受け取る | - |
| F-S09 | パスワードリセット実行 | リセット用URL経由で新しいパスワードを設定する | - |
| F-S10 | マイページ閲覧 | 貸出中・予約中・順番待ち・図書館間貸出の状況を確認 | - |
| F-S11 | 予約キャンセル（自校） | 自校資料の予約をキャンセルする | - |
| F-S12 | 予約キャンセル（図書館間貸出） | 図書館間貸出の予約をキャンセルする | - |
| F-S13 | 他校資料予約 | 他校蔵書に図書館間貸出予約を入れる | UC-12 |

### システム自動処理

| 機能ID | 機能名 | 概要 | 対応UC |
|--------|--------|------|--------|
| F-SYS01 | 予約準備通知メール | 予約資料が準備できた際にメール送信 | UC-03,UC-14 |
| F-SYS02 | 返却リマインドメール | 返却予定日3日前にメール送信 | - |
| F-SYS03 | 予約自動キャンセル | 準備完了から7日以内に貸出なければキャンセル | UC-04 |
| F-SYS04 | 図書館間貸出締め切り処理 | 毎週金曜15:00に他校予約を締め切る | UC-12 |
| F-SYS05 | 貸出上限チェック | 貸出・予約時に同時貸出5冊上限を検証（API・フロント両方） | - |
| F-SYS06 | 延滞制限チェック | 貸出・予約時に延滞中資料の有無を検証（API・フロント両方） | - |

## 課題2（拡張）機能候補

以下はフェーズE（改善・追加実装）での実装対象として検討する。

| 機能ID | 機能名 | 概要 |
|--------|--------|------|
| F-EX01 | 貸出統計ダッシュボード | 人気資料・貸出数の可視化 |
| F-EX02 | 一括アカウント登録 | CSVインポートによる新入生一括登録 |
| F-EX03 | 資料レコメンド | 貸出履歴に基づくおすすめ資料表示 |
| F-EX04 | 延長申請 | 貸出期限の延長をオンラインで申請 |
| F-EX05 | 資料リクエスト | 未所蔵資料の購入リクエスト機能 |
| F-EX06 | Azure EntraID連携 | 学校のMicrosoftアカウントでSSOログイン |
| F-EX07 | 権限テーブル方式への移行 | is_committeeフラグを権限テーブルに移行し、柔軟な権限管理を実現 |
