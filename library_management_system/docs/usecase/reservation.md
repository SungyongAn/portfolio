# ユースケース：予約

---

### UC-29：資料予約（在庫あり）
- **アクター**：生徒、図書委員、司書
- **目的**：利用可能な資料を取り置き予約する
- **前提条件**：
  - 対象資料が「利用可能」である
- **基本フロー**：
  1. 資料詳細画面で「予約する」を選択
  2. システムが資料状態を確認
  3. 問題なければ予約を登録
  4. 司書が資料を予約専用棚へ移動
  5. 利用者へ「準備完了」メール通知
  6. 利用者が来館し貸出処理を実行
- **代替フロー**：
  - 他校資料の場合：図書館間貸出（interlibrary）へ誘導
- **例外**：
  - 同一利用者が既に予約済みの場合：重複予約不可
  - 貸出制限中の利用者：予約不可
- **関連API**：
  - POST /reservations
- **関連画面**：
  - SCR-06（資料詳細）
  - SCR-07（予約一覧）

### UC-30：順番待ち登録（貸出中資料）
- **アクター**：生徒、図書委員、司書
- **目的**：貸出中資料に対して順番待ちキューに登録する
- **前提条件**：
  - 対象資料が貸出中である
- **基本フロー**：
  1. 資料詳細画面で「順番待ち登録」を選択
  2. システムがキューに追加（末尾）
  3. 現在の待ち人数・順位を表示
  4. 登録完了
- **代替フロー**：
  - 他校資料の場合：図書館間貸出へ誘導
- **例外**：
  - 同一利用者が既にキュー内にいる場合：登録不可
- **関連API**：
  - POST /reservations/waitlist
- **関連画面**：
  - SCR-06（資料詳細）

---

### UC-31：予約繰り上げ通知
- **アクター**：システム
- **目的**：返却された資料を次の利用者に割り当てる
- **前提条件**：
  - 順番待ちキューが存在する
- **基本フロー**：
  1. 資料返却を検知（UC-27と連携）
  2. キュー先頭の利用者を取得
  3. 当該利用者の予約を確定状態に更新
  4. 「準備完了」メール通知を送信
  5. 司書が予約棚へ移動
- **例外**：
  - 利用者が貸出制限状態：次順位へスキップ
- **関連API**：
  - PATCH /reservations/{id}/activate（内部処理）
- **関連画面**：
  - （なし：バックエンド処理）

### UC-32：予約受取期限切れ処理
- **アクター**：システム
- **目的**：一定期間受け取られない予約を自動キャンセルする
- **前提条件**：
  - 予約が「準備完了」状態である
- **基本フロー**：
  1. 予約から一定期間（例：7日）経過を検知
  2. 該当予約を自動キャンセル
  3. 次の順番待ち利用者がいる場合、UC-31を実行
  4. 利用者へキャンセル通知を送信
- **例外**：
  - 次の予約者がいない場合：資料を通常棚へ戻す
- **関連API**：
  - PATCH /reservations/{id}/cancel（バッチ処理）
- **関連画面**：
  - （なし：バックエンド処理）
