# ユースケース：司書業務

---

### UC-23：資料追加
- **アクター**：司書
- **目的**：新しい資料を蔵書として登録する
- **前提条件**：
  - 司書がログイン済みである
- **基本フロー**：
  1. 資料管理画面で「新規登録」を選択
  2. バーコードをスキャン（または手入力）
  3. 書誌情報（タイトル・著者・ISBN・NDC分類等）を入力
  4. 所蔵校・分類を確認
  5. 登録を実行
  6. 資料が蔵書として登録される
- **例外**：
  - 必須項目未入力の場合、登録不可
  - 同一バーコードが既に存在する場合、登録不可
- **関連API**：
  - POST /books
- **関連画面**：
  - SCR-09（資料管理）

---

### UC-24：資料情報更新
- **アクター**：司書
- **目的**：既存資料の情報を修正・更新する
- **前提条件**：
  - 対象資料が存在する
- **基本フロー**：
  1. 資料管理画面で対象資料を検索・選択
  2. 編集画面を開く
  3. 書誌情報（タイトル・著者・分類など）を修正
  4. 更新を実行
  5. 情報が更新される
- **例外**：
  - 不正な入力値の場合、更新不可
- **関連API**：
  - PUT /books/{id}
- **関連画面**：
  - SCR-09（資料管理）

---

### UC-25：資料廃棄
- **アクター**：司書
- **目的**：不要になった資料を利用不可状態にする
- **前提条件**：
  - 対象資料が存在する
- **基本フロー**：
  1. 資料管理画面で対象資料を検索・選択
  2. 「廃棄」操作を選択
  3. システムが貸出・予約状況を確認
  4. 問題なければ廃棄処理を実行
  5. 資料の状態が「廃棄」に更新される
- **代替フロー**：
  - 貸出中または予約中の場合：警告表示し、対応後に再実行
- **例外**：
  - システム制約により廃棄不可の場合
- **関連API**：
  - DELETE /books/{id} または PATCH /books/{id}（status変更）
- **関連画面**：
  - SCR-09（資料管理）
