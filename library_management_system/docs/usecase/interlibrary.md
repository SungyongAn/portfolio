# ユースケース：図書館間貸出

---

### UC-17：図書館間貸出予約
- **アクター**：生徒、図書委員（代理）、司書（代理）
- **目的**：他校に所蔵されている資料を取り寄せるために予約する
- **前提条件**：
  - 利用者がログイン済みである
  - 対象資料が他校に所蔵されている
- **基本フロー**：
  1. 資料検索結果から他校所蔵の資料を選択
  2. 「取り寄せ予約」ボタンをクリック
  3. システムが受付可能期間内かを判定（毎週金曜15:00締め切り）
  4. 問題なければ予約を登録
- **代替フロー**：
  - 締切後の場合：次回受付として予約扱いにする or エラー表示
  - 資料が貸出中の場合：返却後に手配される旨を表示
- **例外**：
  - 利用上限に達している場合、予約不可
- **関連API**：
  - POST /interlibrary-reservations
- **関連画面**：
  - SCR-06（資料詳細）

### UC-18：図書館間貸出キャンセル
- **アクター**：生徒、図書委員（代理）、司書（代理）
- **目的**：登録済みの図書館間貸出予約をキャンセルする
- **前提条件**：
  - 対象予約が存在する
  - キャンセル可能状態である（発送前など）
- **基本フロー**：
  1. マイページから予約一覧を表示
  2. 対象予約の「キャンセル」ボタンをクリック
  3. システムが状態を確認
  4. 問題なければキャンセル処理を実行
- **例外**：
  - 発送済みの場合、キャンセル不可
- **関連API**：
  - DELETE /interlibrary-reservations/{id}
- **関連画面**：
  - SCR-07（マイページ）

---

### UC-19：自分の図書館間貸出一覧確認
- **アクター**：生徒、図書委員
- **目的**：自分が予約した図書館間貸出の状況を確認する
- **前提条件**：
  - ログイン済みである
- **基本フロー**：
  1. マイページを開く
  2. 図書館間貸出一覧を表示
  3. 各予約の状態（受付中／発送済／到着済）を確認
- **関連API**：
  - GET /interlibrary-reservations/me
- **関連画面**：
  - SCR-07（マイページ）

---

### UC-20：発送登録
- **アクター**：司書
- **目的**：他校へ送付する資料の発送処理を登録する
- **前提条件**：
  - 対象資料が送付対象として確定している
- **基本フロー**：
  1. 締切処理後、送付対象リストが生成される
  2. 司書が送付リストを確認
  3. 実際に梱包・発送準備を行う
  4. システム上で発送登録を実行
- **関連API**：
  - POST /interlibrary-shipments
- **関連画面**：
  - SCR-10（送付リスト）

---

### UC-21：受取登録
- **アクター**：司書
- **目的**：他校から届いた資料の受取を登録する
- **前提条件**：
  - 発送済みの資料が到着している
- **基本フロー**：
  1. 到着した資料のバーコードをスキャン
  2. システムに受取登録
  3. 状態を「到着済」に更新
  4. 利用者へ通知（メール等）
- **関連API**：
  - POST /interlibrary-receipts
- **関連画面**：
  - SCR-10（受取登録）

---

### UC-22：送付リスト確認
- **アクター**：司書
- **目的**：今週送付する資料の一覧を確認する
- **前提条件**：
  - 締切処理が完了している
- **基本フロー**：
  1. 管理画面を開く
  2. 「送付リスト」を表示
  3. 送付対象資料を確認
- **関連API**：
  - GET /interlibrary-shipments
- **関連画面**：
  - SCR-10（送付リスト）
