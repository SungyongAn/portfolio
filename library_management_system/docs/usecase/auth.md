# ユースケース：認証

---

### UC-01：ログイン
- **アクター**：生徒、司書、管理者
- **目的**：認証情報を用いてシステムにログインする
- **前提条件**：
  - アカウントが存在する
- **基本フロー**：
  1. ログイン画面でメールアドレス・パスワードを入力
  2. システムが認証を実行
  3. 認証成功
  4. ロールに応じたダッシュボードへ遷移
- **例外**：
  - 認証失敗：エラーメッセージ表示
  - アカウントが無効化されている場合：ログイン不可
- **関連API**：
  - POST /auth/login
- **関連画面**：
  - SCR-01（ログイン）

---

### UC-02：パスワードリセット申請
- **アクター**：未ログインユーザー（生徒、司書、管理者）
- **目的**：パスワードを忘れた場合にリセット手続きを開始する
- **前提条件**：
  - メールアドレスが登録されている
- **基本フロー**：
  1. パスワードリセット申請画面を開く
  2. メールアドレスを入力
  3. 送信ボタンをクリック
  4. システムがリセット用トークンを発行
  5. パスワードリセット用URLをメール送信
- **例外**：
  - 未登録メールアドレスの場合：エラー表示（または成功扱いで秘匿）
- **関連API**：
  - POST /auth/password-reset/request
- **関連画面**：
  - SCR-12（パスワードリセット申請）

---

### UC-03：パスワードリセット実行
- **アクター**：未ログインユーザー（生徒、司書、管理者）
- **目的**：リセット用URLを用いて新しいパスワードを設定する
- **前提条件**：
  - 有効なリセットトークンを保持している
- **基本フロー**：
  1. メール内URLからリセット画面を開く
  2. 新しいパスワードを入力（確認含む）
  3. 送信
  4. システムがトークンを検証
  5. パスワードを更新
  6. 完了メッセージ表示
- **例外**：
  - トークン無効・期限切れ：エラー表示
- **関連API**：
  - POST /auth/password-reset/confirm
- **関連画面**：
  - SCR-13（パスワードリセット実行）

---

### UC-04：パスワード変更
- **アクター**：生徒、司書、管理者
- **目的**：ログイン中ユーザーが自身のパスワードを変更する
- **前提条件**：
  - ログイン済みである
- **基本フロー**：
  1. アカウント設定画面を開く
  2. 現在のパスワードを入力
  3. 新しいパスワードを入力（確認含む）
  4. 変更を実行
  5. パスワード更新完了
  6. 完了通知を表示（必要に応じてメール送信）
- **例外**：
  - 現在のパスワード不一致：エラー表示
  - 新パスワードが要件未満：エラー表示
- **関連API**：
  - PUT /auth/password
- **関連画面**：
  - SCR-08（パスワード変更）
