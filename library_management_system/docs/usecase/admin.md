# ユースケース：管理者

---

### UC-26：生徒アカウント登録
- **アクター**：管理者
- **目的**：新規生徒のアカウントを作成する
- **前提条件**：
  - 管理者がログイン済みである
- **基本フロー**：
  1. 管理画面で「新規登録」を選択
  2. メールアドレス・氏名・所属校・学年・クラスを入力
  3. 初期パスワードを設定（または自動生成）
  4. 登録を実行
  5. アカウントが作成される
  6. 初期パスワードを生徒へ通知（手渡し等）
- **例外**：
  - メールアドレスが既に登録済みの場合、登録不可
- **関連API**：
  - POST /users
- **関連画面**：
  - SCR-11（生徒アカウント管理）

### UC-27：生徒アカウント削除
- **アクター**：管理者
- **目的**：不要となった生徒アカウントを削除する
- **前提条件**：
  - 対象生徒が存在する
- **基本フロー**：
  1. 管理画面で生徒一覧を表示
  2. 対象生徒を選択
  3. 「削除」操作を実行
  4. システムが貸出状況を確認
  5. 問題なければ削除処理を実行
- **代替フロー**：
  - 貸出中資料がある場合：警告表示し、対応後に再実行
- **例外**：
  - システム管理上削除不可の場合（管理者自身など）
- **関連API**：
  - DELETE /users/{id}
- **関連画面**：
  - SCR-11（生徒アカウント管理）

### UC-28：生徒一覧取得
- **アクター**：管理者
- **目的**：登録されている生徒の一覧を確認する
- **前提条件**：
  - 管理者がログイン済みである
- **基本フロー**：
  1. 管理画面を開く
  2. 生徒一覧を表示
  3. 検索・フィルタ（学校・学年など）で絞り込み
- **関連API**：
  - GET /users
- **関連画面**：
  - SCR-11（生徒アカウント管理）

---

### UC-29：生徒詳細取得
- **アクター**：管理者
- **目的**：特定の生徒の詳細情報を確認する
- **前提条件**：
  - 対象生徒が存在する
- **基本フロー**：
  1. 生徒一覧から対象生徒を選択
  2. 詳細画面を表示
  3. 基本情報（氏名・所属・状態など）を確認
- **関連API**：
  - GET /users/{id}
- **関連画面**：
  - SCR-11（生徒アカウント管理）
