# ユースケース：貸出・返却

---

### UC-26：資料貸出
- **アクター**：生徒、図書委員、司書
- **目的**：資料を利用者に貸し出す
- **前提条件**：
  - 利用者が貸出可能な状態である（貸出上限未超過など）
  - 対象資料が「貸出可能」状態である
- **基本フロー**：
  1. 貸出モードで利用者を特定（学生証またはID入力）
  2. 資料バーコードをスキャン
  3. システムが予約状況・貸出状態を確認
  4. 問題なければ貸出期限（例：2週間）を設定
  5. 貸出を確定
  6. 返却予定日を画面表示・メール通知
- **代替フロー**：
  - 他利用者の予約が存在する場合：貸出不可としてエラー表示
- **例外**：
  - 利用者が貸出上限を超えている場合：貸出不可
  - 延滞資料がある場合：貸出制限
- **関連API**：
  - POST /loans
- **関連画面**：
  - SCR-10（貸出・返却）

---

### UC-27：資料返却
- **アクター**：生徒、図書委員、司書
- **目的**：貸出中の資料を返却し、利用可能状態に戻す
- **前提条件**：
  - 対象資料が貸出中である
- **基本フロー**：
  1. 返却モードで資料バーコードをスキャン
  2. システムが貸出情報を特定
  3. 返却処理を実行
  4. 資料状態を「利用可能」に更新
  5. 次予約がある場合、予約者へ通知
  6. 資料を返却棚または予約棚へ移動
- **代替フロー**：
  - 他校資料の場合：返送処理フローへ連携（interlibrary）
- **例外**：
  - 対象資料が貸出状態でない場合：エラー表示
- **関連API**：
  - POST /returns または PATCH /loans/{id}/return
- **関連画面**：
  - SCR-10（貸出・返却）

---

### UC-28：貸出延長
- **アクター**：図書委員、司書
- **目的**：貸出期限を延長する
- **前提条件**：
  - 対象資料が貸出中である
  - 延長可能条件を満たしている（予約なし・延滞なし等）
- **基本フロー**：
  1. 利用者がマイページまたは管理画面で対象資料を選択
  2. 「延長」を実行
  3. システムが予約状況を確認
  4. 問題なければ新しい返却期限を設定
  5. 延長完了を表示
- **代替フロー**：
  - 予約がある場合：延長不可
- **例外**：
  - 延長回数上限を超えている場合：延長不可
- **関連API**：
  - PATCH /loans/{id}/extend
- **関連画面**：
  - SCR-11（マイページ）
