# API設計

## 基本方針

- ベースURL：`/api`
- 認証：JWT Bearer Token（Access Token）
- 全エンドポイントは認証必須（`/auth`配下を除く）

---

## HTTPメソッド方針

| メソッド | 用途 |
|----------|------|
| GET | データ取得 |
| POST | 新規作成 |
| PUT | 全更新 |
| PATCH | 部分更新 |
| DELETE | 削除（論理削除含む） |

---

## ステータスコード方針

| コード | 意味 |
|--------|------|
| 200 | 成功 |
| 201 | 作成成功 |
| 400 | リクエスト不正 |
| 401 | 未認証 |
| 403 | 権限なし |
| 404 | リソースなし |
| 500 | サーバーエラー |
