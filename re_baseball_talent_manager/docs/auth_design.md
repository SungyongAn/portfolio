# 認証設計書（auth_design.md）

---

## 1. 認証方式概要

本システムは **メールアドレス + パスワード認証** を採用する。  
認証後は **JWT を利用したトークンベース認証** を行う。

- 認証方式：JWT認証
- パスワードハッシュ：Argon2
- メールアドレス：ユニーク制約
- MFA：将来拡張想定（未実装）

---

## 2. 認証フロー

### 2.1 ログインフロー

1. ユーザーがメールアドレス・パスワードを入力
2. サーバー側でメールアドレス検索
3. Argon2でパスワード検証
4. 認証成功時：
   - アクセストークン発行
   - リフレッシュトークン発行
5. クライアントへ返却

---

## 3. パスワード設計

- ハッシュアルゴリズム：Argon2
- 平文保存：不可
- 初期パスワードはコーチ / 監督が設定
- 初回強制変更：PoCでは未実装

---
## 4. トークン設計

### 4.1 アクセストークン

- 保存場所：フロントエンドメモリ（Vue state 等）
- localStorage / sessionStorage には保存しない
- 有効期限：短時間（例：15分）
- APIリクエスト時に Authorization ヘッダーへ付与

### 4.2 リフレッシュトークン

- 保存場所：HttpOnly Cookie
- サーバー側管理：なし（PoCでは未実装）
- 有効期限：長時間（例：7日）
- 失効管理：Cookie削除のみ（即時失効不可）

---

## 5. ログアウト設計

1. HttpOnly CookieのリフレッシュトークンをCookie削除
2. フロントメモリのアクセストークンを削除 

---

## 6. セキュリティ方針（PoC範囲）

| 項目 | 内容 |
|------|------|
| パスワード保存 | Argon2ハッシュ |
| メール | ユニーク制約 |
| MFA | 将来追加可能設計 |
| リフレッシュトークンDB保存 | HttpOnly Cookieのみ（DB・キャッシュ保存なし） |
| 即時強制失効 | 未対応 |

---
