# 画面一覧

## 画面一覧サマリ（画面数・遷移の概要）
- 画面数：5（viewsのルーティング単位）
- 簡易画面の概要
```
src/
├── components/
│   ├── measurement/
│   │   ├── MeasurementFilterBar.vue（測定記録フィルタバー）
│   │   └── MeasurementTable.vue（測定記録テーブル）
│   ├── member/
│   │   ├── MemberFilterBar.vue（部員フィルタバー）
│   │   ├── MemberTable.vue（部員テーブル）
│   │   └── MemberConfirmModal.vue（部員操作確認モーダル）
│   ├── visualization/
│   │   ├── TrendChart.vue（ECharts折れ線グラフ描画専用）
│   │   ├── TrendChartView.vue（推移分析コンテナ）
│   │   ├── RadarChart.vue（EChartsレーダーチャート描画専用）
│   │   └── RankingTable.vue（順位表示専用）
│   ├── MemberManagement.vue（部員の管理画面、コーチ / 監督）
│   ├── MemberCreate.vue（部員作成、コーチ / 監督）
│   ├── MemberRetire.vue（退部・引退処理、コーチ / 監督）
│   ├── MeasurementResultReview.vue（測定結果の確認/承認画面、部員 / コーチ）
│   ├── MeasurementResultList.vue（測定記録の閲覧、全アカウント）
│   ├── Pagination.vue（ページネーションUIコンポーネント）
│   └── AppHeader.vue（ログイン後の共有ヘッダー）
└── views/
    ├── LoginView.vue（ログイン画面）
    ├── shared/
    │   ├── DashboardView.vue（部員 / コーチ / 監督）
    │   └── ChartView.vue（可視化ダッシュボード、全アカウント）
    └── manager/
        ├── DashboardView.vue（マネージャー）
        ├── MeasurementResultSubmit.vue（測定結果の入力/承認依頼）
        └── MeasurementStatusList.vue（承認依頼中のステータス確認）
```

  - MemberManagement.vue（部員の管理画面、コーチ / 監督）

  - 通知受信時にデータを自動更新（WebSocket連携）
- **遷移先追加**
  - ChartView.vue（可視化ダッシュボード、全アカウント）

### manager/MeasurementResultSubmit.vue（測定結果の入力/承認依頼）

### ■ 課題1
- **対象ロール**：マネージャー
- **主な表示項目**
  - 部員選択（ドロップダウン）
  - 測定日入力
  - 走力（2種）：50m走（sec）/ ベースランニング（sec）
  - 肩力（2種）：遠投（m）/ ストレート球速（km/h）
  - 打力（2種）：打球速度（km/h）/ スイング速度（km/h）
  - 筋力（2種）：ベンチプレス（kg）/ スクワット（kg）
  - 確認ボタン
- **主な入力項目**
  - 測定日（カレンダーから選択 or 手入力）
  - 部員名（ドロップダウン）
  - 各測定項目の数値

### ■ 課題2（実装済み）
- **追加機能**
  - 確認モーダルによる登録フロー（入力内容確認後に承認依頼を送信）
  - 重複登録防止（同一部員・同一計測日のチェック）
  - `rejected` レコードの上書き更新対応
  - エラー種別に応じたインラインメッセージ表示

- **遷移先**
  - 確認モーダル → 承認依頼送信 → 同画面に留まりフォームリセット
