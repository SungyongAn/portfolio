# system_design.md

# システム設計書

## 1. 概要

本ドキュメントは、`re_quest1` におけるシステム全体設計の概要を定義する。

本システムは、開発案件の申請・承認・進捗管理・予算管理を一元化し、案件の予実管理を可視化するWebアプリケーションである。

既存PoCである `quest_1` をベースとし、React + TypeScript / ASP.NET Core Web API にて再実装を行う。

---

## 2. システム構成

```text
[ Browser ]

    React
      │
      ▼

 ASP.NET Core Web API

      │
      ▼

    MySQL
```

---

## 3. システム目的

本システムの目的は以下である。

- 案件情報の一元管理
- 承認フローの可視化
- 進捗管理の可視化
- 予実管理の可視化
- リスク案件の早期発見
- 属人化の低減

---

## 4. 主要機能

| 機能           | 概要                                   |
| -------------- | -------------------------------------- |
| 認証           | ログイン、JWT認証、ログアウト          |
| 案件管理       | 案件作成、案件編集、案件一覧、案件詳細 |
| 承認管理       | 部門承認、本部承認、却下               |
| タスク管理     | タスク作成、タスク更新、ガントチャート |
| 予算管理       | 予算登録、工数管理、経費管理           |
| 通知機能       | 通知一覧、既読管理、WebSocket通知      |
| ダッシュボード | KPI表示、アラート表示、リスク案件表示  |

## 6. 認証方式

認証方式はJWT認証とする。

```text
Authorization: Bearer <token>
```

---

## 7. 認可方式

API側で権限判定を行う。

主な対策は以下とする。

- ロール制御
- 部門制御
- IDOR対策

詳細は `system/roles.md` を参照する。

---

## 8. 通知方式

リアルタイム通知にはWebSocketを利用する。

```text
WebSocket
```

通知例は以下とする。

- 承認依頼
- 承認完了
- 却下
- 案件着手
- 案件完了
- タスク割当

---

## 9. 移行方針

第1段階では既存PoCの完全移植を行う。

変更しないものは以下とする。

- 画面構成
- API仕様
- DB設計
- ロール設計
- ステータス設計
- 通知仕様
- アラート判定

---

## 10. 今後の拡張候補

移植完了後、以下を検討する。

- HttpOnly Cookie認証
- Refresh Token
- CSRF対策
- CI/CD
- ログ監視
- モバイル対応
- KPI分析機能強化

---

## 11. まとめ

本システムでは、既存PoCの仕様を維持しながら、React + TypeScript / ASP.NET Core Web API 構成へ移植する。

第1段階では改善よりも既存機能の再現を優先し、移植完了後に段階的な改善を行う。
