# roles.md

# ロール設計

## 1. 概要

本ドキュメントは、`re_quest1` におけるロール設計を定義する。

本システムでは、利用者の役割に応じて表示内容および操作可能な機能を制御する。

第1段階では既存 `quest_1` のロール設計を維持する。

---

# 2. ロール一覧

| ロール       | 説明         |
| ------------ | ------------ |
| APPLICANT    | 案件申請者   |
| DEPT_MANAGER | 部門管理者   |
| HQ_MANAGER   | 本部管理者   |
| TASK_MEMBER  | タスク担当者 |

---

# 3. APPLICANT

## 概要

案件を申請する利用者。

---

## 主な業務

- 案件申請
- 自案件確認
- 承認状況確認
- 案件進捗確認

---

| 対象         | 可否 |

# 8. 認可方針

権限制御はフロントエンドとバックエンドの両方で実施する。

---

## フロントエンド

- メニュー制御
- ボタン表示制御
- 画面遷移制御

---

## バックエンド

- APIアクセス制御
- 部門制御
- IDOR対策

---

# 9. IDOR対策

対象データ取得時は必ず権限確認を行う。

```text
対象取得
 ↓

PermissionService
 ↓

ロール確認
 ↓

部門確認
 ↓

許可
または
403
```

---

# 10. PermissionService

ASP.NET Core版では PermissionService を利用する。

主な責務

- 案件アクセス判定
- タスクアクセス判定
- 予算アクセス判定
- 通知アクセス判定

---

# 11. 第1段階方針

既存PoCのロール設計を維持する。

変更しない内容

- ロール数
- ロール名
- 権限制御
- 部門制御
- ダッシュボード表示内容

```text

```

の4ロールを利用する。
