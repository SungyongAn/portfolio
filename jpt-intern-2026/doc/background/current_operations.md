# 現在の運用とPoC企画時の課題点

## ■ 現在の運用

### 申請〜承認

- 案件の新規申請は専用の申請システムで実施
  - 入力項目：案件名・目的・概要・概算予算・担当部門
- 承認フロー：申請者 → 部門管理者 → 本部管理者
  - 承認は一方向のみ
  - 却下された場合は申請者が作り直す
- 申請が承認されると、各部門で個別のExcelシートを作成し開発進捗を管理
  - Excelの管理項目：タスク一覧・担当者・ステータス・期限

### 進捗・予算管理

- 各部門のExcelは定期的に本部へメールで共有
- 本部では各部門のExcelを手作業で統合して全体進捗を把握
- 進捗状況は定期的に申請システムのフォームへ再投稿（二重入力が発生）
  - 再投稿の目的：本部での進捗確認・報告のため
- 予算の消費率も個別のExcelで管理しており、リアルタイムでの追跡が困難

### 現在管理している項目

| 管理区分 | 管理項目 | 管理場所 |
|----------|----------|----------|
| 申請情報 | 案件名 | 申請システム |
| 申請情報 | 目的・概要 | 申請システム |
| 申請情報 | 概算予算（工数・金額） | 申請システム |
| 申請情報 | 担当部門・申請者 | 申請システム |
| 開発進捗 | タスク一覧・担当者 | 部門別Excel |
| 開発進捗 | ステータス・進捗率 | 部門別Excel |
| 開発進捗 | 開始日・期限 | 部門別Excel |
| 開発進捗 | 月ごとの進捗報告 | 部門別Excel |
| 予算管理 | 予算項目・予算額 | 予算管理Excel |
| 予算管理 | 実績額・消費率 | 予算管理Excel |

## ■ PoC企画時の課題点と解決方針

| 課題 | 解決方針 |
|------|----------|
| 各部門のExcelフォーマットが統一されておらずデータ統合が困難 | 新アプリで統一されたデータ入力フォームを提供し、フォーマットの差異を吸収する |
| 進捗報告を申請フォームへ再投稿する運用が二重入力となっている | 新アプリ内で申請から進捗管理まで一気通貫で管理し、再投稿を不要とする |
| PC・タブレットいずれからでも全機能を利用したい | WebシステムとしてPC・タブレット対応のレスポンシブデザインで実装する |
