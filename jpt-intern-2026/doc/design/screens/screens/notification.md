# S-NTF-01：通知UI

## ■ 目的

通知の確認、詳細確認、既読管理を行う。

本PoCでは、通知確認のための専用画面遷移を抑え、PCではPopover、スマホではDrawer、詳細確認はDialogで行う構成とする。

---

## ■ 対象ロール

- APPLICANT
- TASK_MEMBER
- DEPT_MANAGER
- HQ_MANAGER

---

## ■ 業務上の役割

- 承認依頼、却下、着手、完了などのイベント通知を確認する
- 未読通知件数を確認する
- 通知詳細を確認し、必要に応じて既読化する
- 一覧画面やダッシュボードの文脈を維持したまま通知確認を行う

---

## ■ 通知対象イベント

| イベント | 通知先     | 説明         |
| -------- | ---------- | ------------ |
| 案件申請 | 部門管理者 | 一次承認依頼 |
| 部門承認 | 本部管理者 | 最終承認依頼 |
| 却下     | 申請者     | 却下通知     |
| 着手     | 関係者     | 案件開始通知 |
| 完了     | 関係者     | 案件完了通知 |

---

## ■ 表示方式

| デバイス | 通知一覧                  | 通知詳細 |
| -------- | ------------------------- | -------- |
| PC       | サイドバーからPopover表示 | Dialog   |
| スマホ   | ヘッダーからDrawer表示    | Dialog   |

---

## ■ 表示項目

| 項目       | 説明                           |
| ---------- | ------------------------------ |
| タイトル   | 通知タイトル                   |
| メッセージ | 通知本文                       |
| 通知種別   | 承認依頼・却下・着手・完了など |
| 対象案件   | 関連案件                       |
| 作成日時   | 通知日時                       |
| 既読状態   | 未読 / 既読                    |

---

## ■ 操作

| 操作         | 内容                                        |
| ------------ | ------------------------------------------- |
| 通知一覧表示 | PCはPopover、スマホはDrawerで通知一覧を表示 |
| 通知詳細表示 | 通知選択時にDialogを表示                    |
| 既読化       | Dialog内で「既読にする」を実行              |
| 全既読       | 通知一覧から全通知を既読化                  |
| 閉じる       | Popover / Drawer / Dialogを閉じる           |

---

## ■ UI制御

### PC

- サイドバーから通知一覧をPopover表示する
- 通知一覧を表示したまま、通知詳細をDialogで確認できる
- Dialogを閉じても通知一覧の文脈を維持する

### スマホ

- ヘッダーから通知一覧をDrawer表示する
- 通知選択時はPC同様にDialogで詳細表示する
- 小さい画面でも通知一覧と詳細確認を分離し、可読性を確保する

### 通知詳細Dialog

- 長文メッセージや承認・却下コメントを全文確認できる
- Dialog内から既読化できる
- 一覧表示を閉じずに詳細確認できるため、複数通知の確認がしやすい

---

## ■ WebSocket通知

- 承認・却下・着手・完了などのイベント発生時にリアルタイム通知を受信する
- 未読件数を同期する
- 画面右下にトースト通知を表示する
- WebSocket切断時は自動再接続を行う
- 接続は認証状態に応じて開始・停止する

---

## ■ バリデーション / 制御

- 未認証ユーザーは通知を取得できない
- ユーザーごとに自分宛の通知のみ取得可能
- 既読化は対象通知の受信者のみ実行可能
- 通知詳細Dialogで未読通知を既読化できる
- 通知種別に応じて表示色を切り替える

---

## ■ 画面遷移

通知確認では専用ページ遷移を行わず、以下のUIで表示する。

| 操作         | 表示方式                     |
| ------------ | ---------------------------- |
| 通知一覧確認 | PC：Popover / スマホ：Drawer |
| 通知詳細確認 | Dialog                       |
| 既読操作     | Dialog内操作                 |

---

## ■ 使用API

| メソッド | エンドポイント          | 説明         |
| -------- | ----------------------- | ------------ |
| GET      | /notifications          | 通知一覧取得 |
| POST     | /notifications/read     | 通知既読化   |
| POST     | /notifications/read-all | 全通知既読化 |

## ■ 備考

- 本PoCではアプリ内通知を主対象とし、メール通知は将来拡張とする
- 画面遷移を減らすことで、通知確認後に元の業務へ戻りやすいUIを目指す
- 「既読＝対応完了」とは限らないため、再未読化や要対応管理は今後の改善候補とする
