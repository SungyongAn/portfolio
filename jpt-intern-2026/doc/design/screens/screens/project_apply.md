# S-APP-02：案件申請

## ■ 目的

新規開発案件を申請する。

本PoCでは、案件申請を専用ページ遷移ではなくDrawerで表示し、ダッシュボードや案件一覧の文脈を維持したまま申請できるUIとする。

---

## ■ 対象ロール

- APPLICANT

---

## ■ 業務上の役割

- 案件申請フローの起点
- 部門管理者への一次承認依頼
- 承認後の進捗管理・予算管理の起点

---

## ■ 表示方式

| 項目         | 内容                                          |
| ------------ | --------------------------------------------- |
| 表示形式     | Drawer                                        |
| 起動元       | サイドバー「案件申請」 / 案件一覧「新規申請」 |
| URL状態      | `/?action=create-project`                     |
| 閉鎖時       | queryを自動削除                               |
| 再読み込み時 | queryに応じてDrawer表示を復元                 |

---

## ■ 入力項目

| 項目             | 必須 | 説明                                 |
| ---------------- | ---- | ------------------------------------ |
| 案件名           | ○    | 50字以内                             |
| 目的・概要       | ○    | なぜこの案件が必要か、何を実現するか |
| 担当部門         | ○    | 申請者の所属部門が自動セット         |
| 概算予算（万円） | ○    | 案件単位の予算額                     |
| 概算工数（人月） | △    | 任意・入力推奨                       |
| 人月単価（万円） | ○    | 案件単位で設定                       |
| 開発開始予定日   | △    | 任意                                 |
| 完了予定日       | △    | 任意                                 |

---

## ■ 操作

| 操作       | 内容                               |
| ---------- | ---------------------------------- |
| 申請       | 案件を申請し、部門管理者に通知する |
| キャンセル | 入力を破棄してDrawerを閉じる       |
| 閉じる     | Drawerを閉じ、URL queryを削除する  |

---

## ■ バリデーション / 制御

- APPLICANTのみ実行可能
- 案件名は50字以内
- 目的・概要は必須
- 概算予算は0より大きい値のみ入力可能
- 人月単価は0より大きい値のみ入力可能
- 完了予定日は開発開始予定日以降の日付のみ入力可能
- 申請後ステータスは `PENDING_DEPT` になる
- 申請成功後、部門管理者へ通知を作成する
- 二重送信を防止する

---

## ■ 画面遷移 / 表示制御

従来の専用ページ遷移ではなく、Drawer表示で申請フォームを開く。

| 操作                       | 表示                                             |
| -------------------------- | ------------------------------------------------ |
| サイドバー「案件申請」選択 | Drawer表示                                       |
| 案件一覧「新規申請」選択   | Drawer表示                                       |
| 申請成功                   | Drawerを閉じ、案件一覧またはダッシュボードを更新 |
| キャンセル                 | Drawerを閉じ、元の画面文脈を維持                 |
| ブラウザ再読み込み         | queryに応じてDrawer表示を復元                    |

---

## ■ URL query制御

| 状態         | URL                             |
| ------------ | ------------------------------- |
| Drawer表示中 | `/?action=create-project`       |
| Drawer閉鎖後 | `/`                             |
| 再読み込み時 | queryを判定してDrawer表示を復元 |

---

## ■ 使用API

| メソッド | エンドポイント | 説明                 |
| -------- | -------------- | -------------------- |
| POST     | /projects      | 案件申請             |
| GET      | /departments   | 部門一覧取得         |
| GET      | /dashboard     | ダッシュボード再取得 |
| GET      | /projects      | 案件一覧再取得       |

---

## ■ 関連機能

- 案件一覧
- ダッシュボード
- 通知機能
- 承認フロー
- 予算管理

---

## ■ 備考

- 本PoCでは、ウォーターフォール型開発案件を前提としている
- 開発手法選択UIは提供しない
- DB上の `development_method` は将来拡張用として保持する
- 案件申請をDrawer化することで、一覧位置やダッシュボードの文脈を維持したまま申請できる
- 申請後は部門承認待ちとなり、部門管理者の一次承認対象になる
