# S-APP-01：案件一覧画面

## ■ 目的

ユーザーが案件の一覧を確認し、対象案件へアクセスする

---

## ■ 対象ロール

- 全ロール

---

## ■ 業務上の役割

案件の検索・参照（進捗管理・承認フローの起点）

---

## ■ 表示項目

| 項目           | 説明                     | 備考                        |
| -------------- | ------------------------ | --------------------------- |
| 案件名         | 案件の名称               | クリックで詳細画面へ遷移    |
| 担当部門       | 所属部門                 |                             |
| 申請者名       | 案件の申請者             |                             |
| ステータス     | 承認状況                 | タグ形式で色分け表示        |
| アラートレベル | 危険・注意の判定         | danger / warning をタグ表示 |
| 進捗率         | タスク進捗の集計         | プログレスバー表示          |
| 予算消化率     | 予算に対する実績額の割合 | プログレスバー表示          |
| 申請日         | 作成日                   |                             |

---

## ■ フィルター・検索

| 項目                     | 説明                     | 備考                    |
| ------------------------ | ------------------------ | ----------------------- |
| キーワード検索           | 案件名の部分一致検索     | デバウンス（300ms）対応 |
| ステータスフィルター     | 複数ステータスの絞り込み | 複数選択可              |
| 部門フィルター           | 部門単位での絞り込み     | HQ_MANAGERのみ表示      |
| アラートレベルフィルター | 危険・注意案件の絞り込み |                         |
| ページネーション         | 1ページあたりの件数切替  | 10・20・50件            |

---

## ■ 操作

| 操作             | 内容               | 条件 |
| ---------------- | ------------------ | ---- |
| 案件選択         | 詳細画面へ遷移     |      |
| フィルタークリア | 検索条件をリセット |      |

---

## ■ バリデーション / 制御

- ロールに応じて表示範囲を制御
  - APPLICANT：自身の案件のみ
  - TASK_MEMBER：自部門の案件のみ
  - DEPT_MANAGER：自部門の案件のみ
  - HQ_MANAGER：全件
- URLクエリパラメータ（`status` / `alertLevel`）による初期フィルター反映に対応

---

## ■ 画面遷移

- 遷移元：ダッシュボード・サイドバー
- 遷移先：
  - 案件詳細画面（S-APP-03）

---

## ■ 使用API

| メソッド | エンドポイント | 説明                                             |
| -------- | -------------- | ------------------------------------------------ |
| GET      | /projects      | 案件一覧取得（ページネーション・フィルター対応） |
| GET      | /departments   | 部門一覧取得（部門フィルター用）                 |

---

## ■ 備考

- 案件申請はダッシュボードの案件申請Drawer（`/?action=create-project`）が主導線
- サーバーサイドページネーション・フィルターに対応
- キーワード検索はデバウンス（300ms）でAPIコールを最適化
