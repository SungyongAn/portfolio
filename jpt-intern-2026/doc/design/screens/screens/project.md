# S-APP-01：案件一覧画面

## ■ 目的
ユーザーが案件の一覧を確認し、対象案件へアクセスする

---

## ■ 対象ロール
- APPLICANT
- DEPT_MANAGER
- HQ_MANAGER

---

## ■ 業務上の役割
案件の検索・参照（進捗管理・承認フローの起点）

---

## ■ 表示項目

| 項目 | 説明 | 備考 |
|------|------|------|
| 案件名 | 案件の名称 | |
| 担当部門 | 所属部門 | |
| 申請者名 | 案件の申請者 | |
| ステータス | 承認状況 | |
| 進捗率 | タスク進捗の集計 | プログレスバー表示 |
| 申請日 | 作成日 | |
| 遅延フラグ | 期限超過の有無 | |

---

## ■ 入力項目
なし

---

## ■ 操作

| 操作 | 内容 | 条件 |
|------|------|------|
| 案件選択 | 詳細画面へ遷移 | |
| 新規申請 | 案件申請画面へ遷移 | APPLICANTのみ |

---

## ■ バリデーション / 制御
- ロールに応じて表示範囲を制御
  - APPLICANT：自身の案件のみ
  - DEPT_MANAGER：自部門
  - HQ_MANAGER：全件

---

## ■ 画面遷移
- 遷移元：ダッシュボード
- 遷移先：
  - 案件詳細画面（S-APP-03）
  - 案件申請画面（S-APP-02）

---

## ■ 使用API

| メソッド | エンドポイント | 説明 |
|----------|----------------|------|
| GET | /projects | 案件一覧取得 |
