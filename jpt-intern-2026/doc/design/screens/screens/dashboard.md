# S-CMN-01：ダッシュボード

## ■ 目的

ユーザーのロールに応じて、案件・承認・通知・KPI情報を集約表示し、主要業務の起点とする。

本PoCでは、画面遷移を抑え、ダッシュボード上で一覧確認・詳細確認・判断・操作へ進めるUIを重視する。

---

## ■ 対象ロール

- APPLICANT
- TASK_MEMBER
- DEPT_MANAGER
- HQ_MANAGER

---

## ■ 業務上の役割

- ロール別の業務起点
- 承認待ち・進行中案件・危険案件などの確認
- KPIカードから対象一覧への即時切替
- 部門別・案件別の状況把握
- 通知確認・案件申請など主要操作への導線

---

## ■ 表示項目

| ------------------ | ------------------------------------ | ------------------------- |
| 未読通知件数       | 未確認通知数                         | 全ロール                  |

| 進行中案件数       | 進行中の案件数                       | ロール別に表示範囲制御    |
| 危険案件数         | アラートレベルdangerの案件数         | DEPT_MANAGER / HQ_MANAGER |

| 部門別KPI          | 部門ごとの案件数・予算・期限超過状況 | HQ_MANAGER                |
| 案件一覧           | KPIカード選択時に対象案件を表示      | 画面内パネル切替          |

| メンバータスク状況 | 部門内メンバーのタスク状況           | DEPT_MANAGER              |

| ------------ | ------------------------------------------------------------ |
| APPLICANT    | 自分の申請案件、承認状況、進行中案件、予算状況               |
| TASK_MEMBER  | 自部門案件、自分の担当タスク、タスク進捗                     |

## ■ 操作

| 操作                   | 内容                                        | 条件               |
| ---------------------- | ------------------------------------------- | ------------------ |
| KPIカード選択          | 対象案件一覧へ切替                          | ロール別に表示制御 |
| 部門タブ選択           | 対象部門のKPI・案件一覧へ切替               | HQ_MANAGER         |
| 案件一覧表示           | ダッシュボード内に案件一覧を表示            | 画面遷移なし       |
| 担当タスク一覧表示     | ダッシュボード内に担当タスク一覧を表示      | TASK_MEMBER        |
| メンバータスク状況表示 | 部門内メンバーの負荷・進捗を表示            | DEPT_MANAGER       |
| 新規案件申請           | 案件申請Drawerを表示                        | APPLICANT          |
| 通知確認               | PCはPopover、スマホはDrawerで通知一覧を表示 | 全ロール           |

---

## ■ UI制御

### KPIカードによる一覧切替

- KPIカード選択時、画面遷移せずダッシュボード内に対象一覧を表示する
- KPIカードは常時表示し、一覧確認中も全体状況を把握できる
- 「戻る」操作により初期ダッシュボード表示へ復帰できる
- 一覧表示状態はコンポーネント内部状態で管理する

### サイドバー導線統一

- サイドバー「案件一覧」選択時も、ダッシュボード内の案件一覧表示へ切り替える
- activeMenu と表示状態を同期し、導線を統一する
- 専用ページ遷移を減らし、ダッシュボード中心の操作体験とする

### 案件申請Drawer

- サイドバー「案件申請」選択時に案件申請Drawerを表示する
- URL query `/?action=create-project` と状態同期する
- Drawer閉鎖時は query を自動削除する
- ブラウザ再読み込み時も案件申請Drawerの表示状態を復元可能とする

---

## ■ バリデーション / 制御

- ロールごとに表示項目を制御する
- ユーザーの所属部門に応じて表示対象データを制御する
- APPLICANTは自分の申請案件のみ表示する
- TASK_MEMBERは自部門案件および担当タスクを表示する
- DEPT_MANAGERは自部門案件・自部門メンバー情報を表示する
- HQ_MANAGERは全部門案件・部門別KPIを表示する
- 権限外データはフロントエンド表示制御およびバックエンドAPI認可で二重に制御する

## ■ 画面遷移

従来の専用画面遷移を抑え、以下の操作は可能な限りダッシュボード内で完結する。

| --------------- | ---------------------------- |
| KPI対象案件確認 | 画面内パネル切替             |
| 案件一覧確認    | ダッシュボード内一覧表示     |
| 案件申請        | Drawer                       |
| 通知一覧確認    | PC：Popover / スマホ：Drawer |
| 通知詳細確認    | Dialog                       |
| タスク詳細確認  | Dialog                       |

## ■ 使用API

| -------- | ----------------- | ------------------------ |

---

## ■ 備考

- 本画面は各ロールの業務起点となるため、一覧確認・詳細確認・判断・操作までを可能な限り同一画面内で完結できるUIを重視する
- KPI表示はPoC後も継続的に最適化する
- KPIカードによる一覧切替は現在コンポーネント内部状態で管理しており、URL queryによる表示状態保持は今後の拡張候補とする
- ブラウザ更新時の一覧状態復元や共有URLによる状態再現は、今後の実務運用向け拡張として検討する
