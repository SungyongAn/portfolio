# S-TSK-01：タスク登録・更新画面

## ■ 目的
タスクを登録・更新し進捗を管理する

---

## ■ 対象ロール
- APPLICANT

---

## ■ 業務上の役割
進捗管理

---

## ■ 入力項目

| 項目 | 必須 | 説明 |
|------|------|------|
| タスク名 | ○ | |
| 担当者 | ○ | 同一部門メンバーから選択 |
| ステータス | ○ | 未着手 / 進行中 / レビュー中 / 完了 |
| 進捗率 | ○ | 0〜100% |
| 開始日 | ○ | |
| 期限 | ○ | |
| 備考 | — | |

---

## ■ 操作

| 操作 | 内容 |
|------|------|
| 追加 | タスクを新規登録する |
| 更新 | タスク情報を更新する |
| 削除 | タスクを削除する |

---

## ■ バリデーション / 制御
- 承認済み案件のみタスク登録・更新が可能（BR-3）
- APPLICANTのみ操作可能
- 期限は開始日以降の日付のみ入力可能
- 進捗率は0〜100の整数値のみ入力可能

---

## ■ 画面遷移
- 遷移元：案件詳細画面（S-APP-03）
- 遷移先：案件詳細画面（S-APP-03）

---

## ■ 使用API

| メソッド | エンドポイント | 説明 |
|----------|----------------|------|
| POST | /projects/{project_id}/tasks | タスク登録 |
| PUT | /projects/{project_id}/tasks/{task_id} | タスク更新 |
| DELETE | /projects/{project_id}/tasks/{task_id} | タスク削除 |
