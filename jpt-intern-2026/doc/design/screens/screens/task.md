# S-TSK-01：タスク登録画面

## ■ 目的

タスクを登録し進捗管理の基盤を構築する

---

## ■ 対象ロール

- APPLICANT

---

## ■ 業務上の役割

進捗管理（タスク登録の起点）

---

## ■ 入力項目

| 項目        | 必須 | 説明                                |
| ----------- | ---- | ----------------------------------- |
| タスク名    | ○    |                                     |
| 工程名      | ○    | ウォーターフォール工程から選択      |
| 担当者      | ○    | 同一部門メンバーから選択            |
| ステータス  | ○    | 未着手 / 進行中 / レビュー中 / 完了 |
| 進捗率（%） | ○    | 0〜100%                             |
| 開始日      | ○    |                                     |
| 期限        | ○    |                                     |
| 備考        | —    | タスクの説明・メモ                  |

---

## ■ 操作

| 操作 | 内容                 | 条件          |
| ---- | -------------------- | ------------- |
| 登録 | タスクを新規登録する | APPLICANTのみ |
| 戻る | 案件詳細画面へ戻る   |               |

---

## ■ バリデーション / 制御

- APPLICANTのみ操作可能
- 承認済み案件（APPROVED / IN_PROGRESS）のみタスク登録が可能
- 期限は開始日以降の日付のみ入力可能
- 進捗率は0〜100の整数値のみ入力可能
- タスク更新・削除はS-APP-03（案件詳細画面）のインラインDialogで実行する

---

## ■ 画面遷移

- 遷移元：案件詳細画面（S-APP-03）
- 遷移先：案件詳細画面（S-APP-03）

---

## ■ 使用API

| メソッド | エンドポイント                    | 説明                                 |
| -------- | --------------------------------- | ------------------------------------ |
| GET      | /projects/{project_id}            | 案件情報取得                         |
| GET      | /users/department/{department_id} | 部門メンバー一覧取得（担当者選択用） |
| POST     | /projects/{project_id}/tasks      | タスク登録                           |

---

## ■ 備考

- タスク更新・削除はS-APP-03（案件詳細画面）のTaskDetailDialog（インライン表示）で完結する
- 工程名はウォーターフォール工程定義（調査・要件定義・設計・実装・テスト・リリース）から選択する
- ステータスと進捗率は連動する（DONEで100%・100%でDONEに自動変更）
