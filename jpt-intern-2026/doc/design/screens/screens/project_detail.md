# S-APP-03：案件詳細画面

## ■ 目的
案件の詳細情報・進捗・予算を確認し、各種操作の起点となる

---

## ■ 対象ロール
- 全ロール

---

## ■ 業務上の役割
案件の詳細確認・各種操作の起点

---

## ■ 表示項目

| 項目 | 説明 |
|------|------|
| 案件名 | |
| 目的・概要 | |
| 担当部門 | |
| 申請者名 | |
| 概算工数（人月） | |
| 概算予算額（万円） | |
| 人月単価（万円） | |
| 現在のステータス | |
| 進捗率バー | 配下タスクの加重平均で自動算出 |
| タスク一覧 | タスク名・担当者・ステータス・進捗率・開始日・期限・備考 |
| 予算サマリ | 予算工数・実績工数・工数消化率・予算金額・実績金額・金額消化率・残工数・残予算 |
| 遅延フラグ | 期限超過タスクの有無 |

---

## ■ 操作

| 操作 | 内容 | 条件 |
|------|------|------|
| 承認・却下 | 承認・却下画面へ遷移 | DEPT_MANAGER / HQ_MANAGERのみ・承認待ち案件のみ |
| タスク登録・更新 | タスク登録・更新画面へ遷移 | APPLICANTのみ・承認済み案件のみ |
| 工数実績入力 | 工数実績入力画面へ遷移 | APPLICANTのみ・承認済み案件のみ |
| 直接経費入力 | 直接経費入力画面へ遷移 | APPLICANTのみ・承認済み案件のみ |

---

## ■ バリデーション / 制御
- ロールに応じてアクセス可能な案件を制御
  - APPLICANT：自身の案件のみ
  - DEPT_MANAGER：自部門の案件のみ
  - HQ_MANAGER：全部門の案件

---

## ■ 画面遷移
- 遷移元：案件一覧画面（S-APP-01）
- 遷移先：
  - 承認・却下画面（S-APR-01）
  - タスク登録・更新画面（S-TSK-01）
  - 予算管理画面（S-BDG-01）

---

## ■ 使用API

| メソッド | エンドポイント | 説明 |
|----------|----------------|------|
| GET | /projects/{project_id} | 案件詳細取得 |
| GET | /projects/{project_id}/tasks | タスク一覧取得 |
| GET | /projects/{project_id}/budget | 予算サマリ取得 |
