# S-AUTH-01：ログイン画面

## ■ 目的

ユーザー認証を行い、システム利用を開始する

---

## ■ 対象ロール

- 全ロール

---

## ■ 業務上の役割

認証

---

## ■ 表示項目

| 項目           | 説明       | 備考 |
| -------------- | ---------- | ---- |
| メールアドレス | ログインID |      |
| パスワード     | 認証用     |      |

---

## ■ 入力項目

| 項目           | 必須 | 説明       | 備考 |
| -------------- | ---- | ---------- | ---- |
| メールアドレス | ○    | ユーザーID |      |
| パスワード     | ○    | 認証情報   |      |

---

## ■ 操作

| 操作     | 内容         | 条件 |
| -------- | ------------ | ---- |
| ログイン | 認証処理実行 |      |

---

## ■ バリデーション / 制御

- メール形式チェック
- 未入力チェック
- 認証失敗時エラーメッセージ表示

---

## ■ 画面遷移

- 遷移先：ダッシュボード

---

## ■ 使用API

| メソッド | エンドポイント | 説明     |
| -------- | -------------- | -------- |
| POST     | /auth/login    | ログイン |

---

## ■ 備考

- アクセストークン・リフレッシュトークンともに sessionStorage で管理する
- ブラウザタブを閉じるとセッションが終了する
- XSS攻撃対策として localStorage へのトークン保存は行わない
- WebSocket通知接続時はアクセストークンをURLクエリパラメータで渡す
