# S-APR-01：承認・却下画面

## ■ 目的

案件の承認・却下を行う

---

## ■ 対象ロール

- DEPT_MANAGER
- HQ_MANAGER

---

## ■ 業務上の役割

承認フロー

---

## ■ 表示項目

| 項目       | 説明                 |
| ---------- | -------------------- |
| 案件名     |                      |
| 担当部門   |                      |
| 申請者     |                      |
| 概算工数   |                      |
| 概算予算   |                      |
| 開始予定日 |                      |
| 完了予定日 |                      |
| ステータス | タグ形式で色分け表示 |

---

## ■ 入力項目

| 項目     | 必須        | 説明                        |
| -------- | ----------- | --------------------------- |
| 却下理由 | ○（却下時） | 却下時は必須入力・500字以内 |

---

## ■ 操作

| 操作 | 内容               | 条件                                 |
| ---- | ------------------ | ------------------------------------ |
| 承認 | 承認処理を実行する | 承認待ち案件のみ                     |
| 却下 | 却下処理を実行する | 承認待ち案件のみ・却下理由の入力必須 |

---

## ■ バリデーション / 制御

- 承認待ち案件のみ操作可能
- 却下時は却下理由の入力を必須とする
- 却下理由は500字以内
- DEPT_MANAGERはPENDING_DEPT状態の案件のみ操作可能
- HQ_MANAGERはPENDING_HQ状態の案件のみ操作可能
- 承認・却下ボタンは処理完了後に非活性化し二重送信を防止する

---

## ■ 画面遷移

- 遷移元：案件詳細画面（S-APP-03）
- 遷移先：案件一覧画面（S-APP-01）

---

## ■ 使用API

| メソッド | エンドポイント                 | 説明                                    |
| -------- | ------------------------------ | --------------------------------------- |
| GET      | /projects/{project_id}         | 案件詳細取得                            |
| POST     | /projects/{project_id}/approve | 承認・却下（reject_reasonの有無で判定） |

---

## ■ 備考

- 承認・却下は同一エンドポイント（`/approve`）を使用し、`reject_reason` の有無で判定する
- `reject_reason` が null または空の場合は承認、値がある場合は却下として処理する
