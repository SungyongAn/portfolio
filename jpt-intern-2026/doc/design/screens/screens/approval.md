# S-APR-01：承認・却下画面

## ■ 目的
案件の承認・却下を行う

---

## ■ 対象ロール
- DEPT_MANAGER
- HQ_MANAGER

---

## ■ 業務上の役割
承認フロー

---

## ■ 表示項目

| 項目 | 説明 |
|------|------|
| 案件名 | |
| 目的・概要 | |
| 概算工数（人月） | |
| 概算予算額（万円） | |
| 担当部門 | |
| 申請者名 | |
| 開発開始予定日 | |
| 完了予定日 | |

---

## ■ 入力項目

| 項目 | 必須 | 説明 |
|------|------|------|
| 却下理由 | ○（却下時） | 却下時は必須入力（BR-9） |
| コメント | — | 承認時は任意 |

---

## ■ 操作

| 操作 | 内容 | 条件 |
|------|------|------|
| 承認 | 承認処理を実行する | 承認待ち案件のみ |
| 却下 | 却下処理を実行する | 承認待ち案件のみ・却下理由の入力必須 |

---

## ■ バリデーション / 制御
- 承認待ち案件のみ操作可能
- 却下時は却下理由の入力を必須とする（BR-9）
- DEPT_MANAGERはPENDING_DEPT状態の案件のみ操作可能
- HQ_MANAGERはPENDING_HQ状態の案件のみ操作可能

---

## ■ 画面遷移
- 遷移元：案件詳細画面（S-APP-03）
- 遷移先：案件一覧画面（S-APP-01）

---

## ■ 使用API

| メソッド | エンドポイント | 説明 |
|----------|----------------|------|
| POST | /projects/{project_id}/approval/approve | 承認 |
| POST | /projects/{project_id}/approval/reject | 却下 |
