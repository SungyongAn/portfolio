# ロール権限マトリクス

> 各ロールが操作可能なリソースとアクセス範囲を定義します。

---

## ■ ロール定義

| ロールID     | ロール名   | 説明                                                     |
| ------------ | ---------- | -------------------------------------------------------- |
| TASK_MEMBER  | 担当者     | 各部門のタスク担当者・自部門案件のタスクを更新する       |
| APPLICANT    | 申請者     | 各部門の案件担当者・案件を申請し進捗・予算実績を管理する |
| DEPT_MANAGER | 部門管理者 | 各部門の管理者・自部門の案件を管理する                   |
| HQ_MANAGER   | 本部管理者 | 関連部門全体を管理する本部のスタッフ                     |

---

## ■ データアクセス範囲

| ロール     | 参照可能な案件         | 操作可能な範囲                           |
| ---------- | ---------------------- | ---------------------------------------- |
| 担当者     | 自部門の全案件         | 自部門案件のタスク更新（進行中まで）     |
| 申請者     | 自身が申請した案件のみ | 自身が申請した案件のみ                   |
| 部門管理者 | 自部門の全案件         | 自部門の全案件（承認・却下・着手・完了） |
| 本部管理者 | 全部門の全案件         | 全部門の全案件（承認・却下・着手・完了） |

---

---

## ■ 補足・設計判断

- 開発者ロールは廃止し、申請者がタスク登録・工数実績・直接経費の入力を行う
- タスク更新は申請者・担当者のみ実行可能（部門管理者・本部管理者は確認・承認・着手・完了のみ）
- タスク登録・削除は申請者・部門管理者・本部管理者が実行可能（担当者は不可）
- ロールは1ユーザーにつき1ロールを付与する（兼任は考慮しない）
- 代理承認はPoC範囲では不要
- 着手操作は申請者（自案件のみ）・部門管理者・本部管理者が実行可能（担当者は不可）
